# 人工生命 × 小型大模型演化研究计划

**版本：** v0.1（草案）  
**日期：** 2026-04-02  
**研究性质：** 探索性基础研究  

---

## 一、研究背景与动机

### 1.1 问题的起点

传统人工生命研究（Avida、Tierra）以简单指令集为生命最小单位，验证了数字生命可以在简单规则下演化出真实复杂性。然而这类系统缺乏"内部表示"——每个生命体只是一串规则，没有任何形式的内部状态整合。

近年来，小型语言模型（<1B参数）的成熟为一个新方向打开了窗口：**以具有初步学习和推理能力的小型神经网络作为人工生命的最小单元**，让它们在演化压力下涌现出复杂行为、通信协议乃至更高层次的涌现现象。

### 1.2 相关前沿工作

| 工作 | 核心贡献 | 与本研究的关系 |
|------|----------|----------------|
| **Lilith**（arxiv:2507.04575） | 模块化LLM + 化学信号通信，优化目标为意识涌现 | 直接参照框架，但本研究聚焦演化而非监督训练 |
| **PD-NCA** | 自然语言反馈调控人工生命演化 | 验证语义信号可以塑造演化方向 |
| **MARL 向量通信** | 多智能体自发演化出有语义的向量信号 | 奠定向量通信方案的可行性基础 |
| **Words Evolution**（arxiv:2505.05863） | 智能体携带LLM词汇，通过竞争演化语言 | 语言涌现的直接先验 |
| **Self-Evolving Agent**（arxiv:2601.11658） | 遗传算法 + 任务失败触发演化 | 演化机制设计参考 |

## 二、核心研究问题

本研究围绕三个层次的问题展开，从工程可行到科学探索再到哲学边界：

### 问题1（工程层）：行为涌现
> **小型神经网络能否通过演化压力，在无监督条件下涌现出协作行为？**

- 是否可行：高度可行，MARL文献有大量支撑
- 实验周期：2~4周可出第一批结果
- 成功标准：智能体在无硬编码协作规则的情况下，演化出比随机行为显著更优的集体觅食/生存策略

### 问题2（科学层）：通信协议自发涌现
> **通信协议（符号/向量的语义结构）能否完全通过演化自发形成，而非人为设计？**

- 是否可行：中等可行，部分MARL工作已有初步证据
- 实验周期：1~3个月
- 成功标准：向量通信中出现可重复识别的语义聚类（如特定维度模式稳定对应特定情境）

### 问题3（哲学+科学层）：意识相关物涌现
> **"意识的功能相关物"能否从多个小模型的交互中涌现？**

- 是否可行：部分可测量，但"难问题"无法从外部证伪
- 实验周期：长期（6个月以上）
- 成功标准：测量到 Φ（信息整合度）、全局工作空间广播、自我/他者区分等指标的涌现

**关键策略：从问题1入手，但在实验设计里埋下通向问题3的观测指标。**

## 三、实验设计

### 3.1 阶段0：最小可验证实验（第1~2周）

**目标：** 验证向量通信下的协作行为涌现

**环境设计：**
- 二维离散格世界（32×32 或 64×64）
- 食物资源随机刷新（刷新率可调）
- 智能体消耗能量存活，达到能量阈值可繁殖
- 繁殖 = 权重复制 + 高斯噪声变异

**智能体架构：**
```
输入层：观测(10维) + 邻居消息聚合(16维) = 26维
隐藏层1：64神经元，ReLU
隐藏层2：64神经元，ReLU
输出：
  ├── 动作头(5维)：上/下/左/右/采集，softmax
  └── 消息头(16维)：广播向量，tanh
总参数量：~6,400（每个智能体）
```

**演化机制：**
- 种群大小：100~500个智能体
- 适应度：存活时间 × 资源收集量
- 选择：锦标赛选择（tournament selection）
- 变异：权重添加 σ=0.01 高斯噪声
- 无梯度优化，纯遗传算法

**观测指标（阶段0）：**
1. 种群平均适应度随代数的变化曲线
2. 消息向量的主成分分布（PCA降维可视化）
3. 消息向量与情境的相关性（是否出现语义分化）

---

### 3.2 阶段1：引入 CoT 轻量化通信（第1~2个月）

**目标：** 探索思维链通信是否带来行为优势

**改动：** 将 MLP 智能体替换为小型语言模型（GPT-2 small 或 Phi-1），使用固定模板思维链：

```
观察到：[环境状态]
收到消息：[邻居消息]
思考：[2~3句推理]
决定：[动作]
发送：[消息摘要]
```

**通信方案对比实验：**

| 组别 | 通信方式 | 目的 |
|------|----------|------|
| 对照组A | 无通信 | 基线 |
| 实验组B | 向量通信（阶段0） | 低级协调 |
| 实验组C | 固定模板CoT通信 | 符号推理协调 |
| 实验组D | 向量+CoT混合 | 混合协调 |

---

### 3.3 阶段2：开放式通信协议演化（第3~6个月）

**目标：** 让通信格式本身也纳入演化，观察语义是否自发涌现

**核心改变：**
- 不再固定词汇表或向量维度
- 通信协议本身（消息格式、长度、编码方式）作为基因的一部分参与演化
- 引入"通信成本"惩罚项，迫使智能体演化出高效的编码方式

**这是最接近"语言起源"问题的实验设计。**

## 四、通向问题3的观测指标

在阶段0~2中，持续记录以下指标，为问题3积累数据：

| 指标 | 理论来源 | 测量方法 | 工具 |
|------|----------|----------|------|
| **信息整合度 Φ** | 整合信息论（IIT） | 计算系统各部分因果贡献 | PyPhi（简化版） |
| **全局工作空间广播** | 全局工作空间理论（GWT） | 追踪特定信号在种群中的传播范围和速度 | 自定义追踪器 |
| **自我/他者区分** | 自我模型理论 | 分析隐藏层对自身状态 vs 外部信息的权重差异 | 激活分析 |
| **时间整合深度** | 记忆与预测 | 分析智能体对历史输入的依赖程度 | 消融实验 |
| **元认知迹象** | 高阶思维 | 智能体是否对自身行为产生"预测误差修正" | 行为分析 |

**重要说明：** 这些指标的涌现不等于意识的出现，但它们是目前科学界认为与意识功能最相关的可测量特征。记录它们是为了建立"可比较的框架"，而非声称实现了意识。

## 五、阶段路线图

```
第1~2周    阶段0：最小实验
           ├── 搭建二维格世界环境
           ├── 实现 MLP 双输出头智能体
           ├── 实现遗传算法演化循环
           └── 可视化：适应度曲线 + PCA消息分布

第1~2个月  阶段1：CoT通信引入
           ├── 集成小型LLM（GPT-2 small）
           ├── 设计固定模板思维链
           ├── 对比实验（4组）
           └── 分析通信方式对适应度的影响

第3~6个月  阶段2：开放式演化
           ├── 通信协议纳入基因组
           ├── 引入通信成本惩罚
           ├── 语义涌现检测
           └── 持续记录问题3指标

第6个月+   阶段3（开放）
           ├── 根据阶段2结果决定方向
           ├── 可能引入更大模型（<1B参数）
           └── 发表/开源
```

## 六、技术栈建议

| 组件 | 推荐工具 | 理由 |
|------|----------|------|
| 演化框架 | Python + NumPy | 轻量，阶段0无需GPU |
| 神经网络 | PyTorch | 阶段1引入LLM时需要 |
| 小型LLM | GPT-2 small（117M）或 Phi-1（1.3B） | 参数适中，可在消费级GPU运行 |
| Φ计算 | PyPhi | IIT标准工具库 |
| 可视化 | Matplotlib + Plotly | 适应度曲线 + 向量空间可视化 |
| 实验管理 | Weights & Biases（W&B） | 追踪多次实验结果 |

## 七、开放问题与风险

### 主要风险
1. **演化停滞**：种群可能陷入局部最优，长时间不出现有趣行为。缓解：调整变异率、增加环境多样性、引入竞争压力。
2. **计算资源**：阶段1引入LLM后计算量大幅上升。缓解：批处理、异步更新、量化推理。
3. **结果解读**：消息向量的"语义"可能是研究者的主观投射。缓解：使用盲评和统计检验。

### 开放问题（值得长期探索）
- 演化出的通信协议是否具有组合性（compositionality）？
- 多代演化后，后代是否会"继承"祖先演化出的通信约定？
- 环境复杂度与涌现现象复杂度之间是否存在相变？
